## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 微型餐飲店（小吃店/咖啡廳） | ~10 萬 | 想用 POS 但商用 iCHEF/肚金月費 1,500-3,000 太貴 |
| 微型零售店（雜貨/文創/3C） | ~8 萬 | 需要簡單結帳、進銷存，但商用 POS 太複雜 |
| 微型服務業（美髮/工作室/維修） | ~5 萬 | 服務計價、技師抽成、客戶記錄，無簡單工具 |
| 連鎖加盟（5-20 店） | ~2,000 | 多店管理、統一商品主檔、現金流彙整 |

### 1.2 為什麼不做替代方案

- **商用 POS（iCHEF/Odoo/POS）**：月費 1,500-5,000 NT$，對微型店家太貴
- **手寫記帳**：月底對帳痛苦、無法追蹤熱賣商品
- **Excel 自製**：可做但介面不友善、無法快速結帳
- **我們的解法**：一套 POS 可切換餐飲/零售/服務 3 個行業模板 + 12 件樣板商品 × 3 行業 = 36 件預載，純前端零月費

---

## 2. 解決方案

### 2.1 核心價值主張

> 「3 種行業模板 + 36 件預載商品 — 開店 5 分鐘就能開始結帳。」

### 2.2 使用者流程

1. 首次開店：選擇行業（餐飲/零售/服務）
2. 系統自動載入對應的 12 件預載商品（餐飲：蛋餅/奶茶/便當；零售：礦泉水/零食/3C；服務：洗髮/美甲/維修）
3. 客製化商品（改名/調價/新增/隱藏）
4. 結帳模式：三欄 POS UI（商品清單/購物車/付款方式）
5. 月底報表：熱賣排行、營業額、技師抽成

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] 3 行業模板（餐飲/零售/服務），預載 12 件商品
- [ ] 商品 CRUD（改名/調價/分類/啟用停用）
- [ ] 三欄 POS UI（商品列表 / 購物車 / 付款）
- [ ] 現金/信用卡/電子支付 3 種付款方式
- [ ] 訂單歷史 + 搜尋
- [ ] 月報表（營業額/熱賣排行/技師抽成）
- [ ] 快照備份（JSON 匯出/匯入）

### 3.2 v2（加值）

- [ ] 多店管理（5-20 店）
- [ ] 進銷存（進貨 + 庫存預警）
- [ ] 會員系統（點數/折扣）
- [ ] LINE 推播電子發票

### 3.3 明確不做

- 線上點餐/電商（純店內 POS）
- 稅務申報（交給會計/國稅局軟體）
- 多幣別（先 NTD）
- 硬體整合（出單印表機/錢櫃）— v3 規劃

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| 商業法規（電子發票/統一編號） | 🟠 中 | v2 加；目前先 PDF 收據 + 明確聲明 |
| localStorage 容量（訂單累積） | 🟠 中 | IndexedDB；舊訂單 90 天後自動歸檔 |
| 多裝置同步需求 | 🟡 低 | v2 加雲端 |
| 價格設定錯誤（商業影響） | 🔴 高 | 結帳前確認彈窗 + 月報表數字對照 |
